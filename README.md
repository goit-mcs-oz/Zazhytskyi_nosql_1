1. Дані аудіохарактеристик є пов'язаними між собою і часто використовуються разом. Таке вкладення є вигідним, оскільки дозволяє отримати всі аудіохарактеристики одним полем. Недоліками є довші звернення до вкладених полів та складніше оновлення окремих вкладених значень.

2. Зберігання виконавців у вигляді масиву полегшує підрахунок і роботу з окремими виконавцями. Також стають простішими запити пошуку та розгортання масиву за допомогою оператора `$unwind`.

3. Оператор `$out` у MongoDB використовується наприкінці Aggregation Pipeline для збереження результатів агрегації в колекцію. Він повністю замінює цільову колекцію результатами агрегації.

   Оператор `$merge` також використовується наприкінці Aggregation Pipeline для запису результатів агрегації в колекцію, але, на відміну від `$out`, не обов'язково замінює всю колекцію. Він може вставляти нові документи, оновлювати наявні або виконувати обидві дії.

   Отже, `$out` використовується, коли потрібно створити або повністю перезаписати колекцію, а `$merge` — коли необхідно оновити існуючу колекцію.


## Частина 2 — Запити до даних

1. $unwind — стадія, яка розгортає масив із документа, створюючи окремий вихідний документ для кожного елемента масиву.
2. $stdDevPop і $stdDevSamp — це оператори MongoDB для обчислення стандартного відхилення. Вони відрізняються тим, для яких даних виконується розрахунок.
$stdDevPop — стандартне відхилення генеральної сукупності.
$stdDevSamp — стандартне відхилення вибірки.
